|||||||||||||||Texture Tool|||||||||||||||||
Пояснительная записка к учебной программе

///1. Назначение программы
Программа написана на Python для учебной задачи по обработке изображений.
Она решает три основные подзадачи:
	1. Конвертация карты высот (height map) в карту нормалей (normal map).
	2. Сборка одного RGBA/aRGB изображения из отдельных чёрно-белых изображений каналов R, G, B и Alpha.
	3. Обратная операция: разбор обычного RGBA изображения на отдельные чёрно-белые каналы R, G, B и A.
Программа имеет графический интерфейс на tkinter и работает с изображениями через библиотеку Pillow.

///2. Какие файлы лежат в папке проекта
В папке проекта находятся:
	4. gircon_tool.py
Это основной файл программы. В нём находится весь код интерфейса, загрузки файлов, обработки изображений, drag-and-drop и режима просмотра.
	5. requirements.txt
Это список внешних библиотек Python, которые нужны для работы программы.
В данном проекте используются:
* Pillow - обработка изображений.
* tkinterdnd2 - поддержка drag-and-drop.
	6. manual.txt
Это данный текстовый файл с описанием структуры проекта и принципов работы программы.
	7. build.bat
Это файл автоматической сборки программы в exe.
Он запускает PyInstaller, собирает итоговый exe, а затем удаляет временный мусор сборки.
	8. texture_tool.spec
Это файл конфигурации PyInstaller.
Он описывает, как именно собирать программу в exe и какие данные нужно включать в сборку.
	9. TextureTool.exe
Это готовый исполняемый файл программы после сборки.
Он появляется в этой же папке рядом с build.bat.

///3. Как запустить программу
	10. Установить Python 3.
	11. Установить зависимости командой:
pip install -r requirements.txt
	12. Запустить программу командой:
python gircon_tool.py
	
///4. Как собрать exe
Для сборки exe нужно запустить файл:
build.bat
	Или выполнить команду:
build.bat
После сборки:
	13. Готовый файл TextureTool.exe лежит в этой же папке проекта.
	14. Временные папки сборки и __pycache__ автоматически удаляются.
	15. В папке остаются только исходники, manual, build.bat, spec и готовый exe.

4. Общая логика работы программы
Программа разделена на 3 вкладки:
	16. Height -> Normal
Из карты высот строится карта нормалей.
	17. BW -> aRGB
Из отдельных чёрно-белых изображений R, G, B и Alpha собирается одно итоговое RGBA изображение.
	18. RGBA -> Channels
Из одного RGBA изображения извлекаются четыре отдельные чёрно-белые карты: R, G, B и A.
Также в программе есть:
	* drag-and-drop загрузка файлов;
	* предпросмотр изображений;
	* полноэкранный просмотр по двойному клику;
	* zoom колесом мыши;
	* перемещение увеличенного изображения удержанием левой кнопки мыши;
	* миникарта и индикатор масштаба в полноэкранном режиме.

5. Объяснение основных частей кода
	5.1. Функция create_normal_map(...)
Эта функция создаёт normal map из карты высот.
Принцип работы:
19. Исходное изображение переводится в оттенки серого.
20. Для каждого пикселя берутся соседние пиксели слева, справа, сверху и снизу.
21. По разнице яркости между соседними пикселями вычисляется наклон поверхности.
22. На основе наклона вычисляется вектор нормали.
23. Вектор переводится в цвет RGB normal map.
24. При необходимости сохраняется alpha-канал исходного изображения.
Переключение OpenGL / DirectX меняет знак зелёного канала normal map.
	5.2. Функция build_argb_image(...)
Эта функция собирает итоговое RGBA изображение из отдельных каналов.
Принцип работы:
25. Канал R берётся как базовый размер.
26. Каналы G, B и Alpha при необходимости автоматически подгоняются к тому же размеру.
27. Если alpha не загружен отдельным файлом, он может создаваться из постоянного значения ползунком.
28. Alpha можно инвертировать и дополнительно усиливать/ослаблять через ползунок прозрачности.
29. Итог собирается командой merge в одно RGBA изображение.
	5.3. Функция split_rgba_image(...)
Эта функция выполняет обратную операцию.
Принцип работы:
30. Исходное изображение приводится к формату RGBA.
31. Из него извлекаются четыре отдельных канала.
32. Каждый канал возвращается как отдельное чёрно-белое изображение.
33. Затем эти каналы можно сохранить все сразу в выбранную папку.
	5.4. Класс PreviewCard
Этот класс отвечает за карточки предпросмотра.
Он выполняет несколько задач:
* показывает изображение;
* показывает краткую информацию о размере и формате;
* хранит текущее изображение для повторного открытия;
* открывает полноэкранный viewer по двойному клику.
	5.5. Класс ScrollablePanel
Этот класс нужен для левой панели интерфейса.
Так как настроек и кнопок много, без прокрутки нижняя часть могла не помещаться на маленьких экранах.
Поэтому левая панель сделана прокручиваемой.
	5.6. Класс ImageViewer
Этот класс открывает изображение во весь экран.
Что он делает:
34. Показывает изображение отдельно от основного окна.
35. Позволяет увеличивать и уменьшать изображение колесом мыши.
36. Позволяет перемещать увеличенное изображение удержанием левой кнопки мыши.
37. Показывает текущий масштаб.
38. Показывает миникарту, чтобы видеть положение текущей области просмотра.
Почему был сделан отдельный viewer:
В обычной карточке предпросмотра изображение маленькое, а для проверки деталей текстур нужен режим увеличения.
	5.7. Класс GirconTool
Это основной класс программы.
Он:
	* создаёт окно;
	* создаёт вкладки;
	* хранит текущее состояние загруженных изображений;
	* обрабатывает нажатия кнопок;
	* запускает конвертацию и сохранение файлов;
	* обрабатывает drag-and-drop.

6. Почему код сделан в одном основном файле
Для учебной задачи выбран простой и понятный вариант:
39. Код легче читать преподавателю без переходов между многими файлами.
40. Легче показать, где интерфейс, где логика обработки и где сохранение.
41. Проект остаётся компактным и его проще запускать.

7. Как работает сохранение каналов в обратном режиме
Во вкладке RGBA -> Channels пользователь:
42. Загружает исходное RGBA изображение.
43. Нажимает "Извлечь каналы".
44. Нажимает "Сохранить все каналы".
45. Выбирает папку.
46. Программа сохраняет 4 файла разом:
	* prefix + имя_исходника + _R.png
	* prefix + имя_исходника + _G.png
	* prefix + имя_исходника + _B.png
	* prefix + имя_исходника + _A.png
Пример:
Если исходный файл называется wall.png, а префикс равен split_, то будут созданы:
	* split_wall_R.png
	* split_wall_G.png
	* split_wall_B.png
	* split_wall_A.png
8. Почему в программе используется Pillow
Pillow выбрана потому что:
47. Это простая и стандартная библиотека для Python.
48. Она хорошо подходит для учебных задач.
49. Она умеет читать, изменять, объединять и сохранять изображения в нужных форматах.
9. Итог
Программа реализует все требуемые функции учебной задачи:
* height map -> normal map;
* 3 чёрно-белых карты + alpha -> одно RGBA изображение;
* RGBA изображение -> отдельные каналы R, G, B, A;
* удобный интерфейс;
* drag-and-drop;
* просмотр результата в полноэкранном режиме.
* сборка в exe через build.bat.
Проект сделан специально в понятном и достаточно простом стиле, чтобы его можно было легко объяснить и защитить в институте.
